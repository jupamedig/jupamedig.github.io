Inline vs Inline-Block: What's the Difference?

First of all, let's define an element. An html page is composed of elements. An element is a piece of code that starts with an opening tag and ends with a closing tag. You can think of each element as a rectangular box. Traditionally, elements have either been defined as inline elements or block elements.

Inline Elements

Inline elements can exist on a line of text and next to other inline elements. They don't interrupt the flow of text whatsoever. But they can't be assigned a height or a width.

Examples of inline elements include <span>, <strong>, and <a>.

Block Elements

Block elements, on the other hand, by default take up as much horizontal space as they can. They can't exist on a line with text or any other elements on it (unless the block element gets "floated," but that's a separate conversation). However unlike inline elements, you can specify their height and width.

Some commonly used block elements are <p>, <h1>, <div> and <section>.

Inline-Block Elements

And then there's the inline-block element. Like an inline element, it's able to stay on the same row with text and other inline elements. And like a block element, it can take a specified height and width.

To make an element inline-block, you just have to give its CSS display property the value of "inline-block". Similarly, you can turn an inline element into a block element just by setting the corresponding CSS display value to "block".
