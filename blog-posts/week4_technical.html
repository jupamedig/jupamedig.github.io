map { |obj| block }

The main idea behind "enumerable" in Ruby, I think, is that it enables us to access individual elements in a collection and do something to them if we so choose.

#map is an enumerator in the enumerable module for this reason. When applied to a collection of elements, it will execute a block of code, if specified, on each element and return a new array of the resulting elements.

Let's take a look at some examples...

example 1:

(1..5).map { |i| i**2 }

# => [1, 4, 9, 16, 25]

example 2:

["a","b","c","d","e"].map { |i| i*3 }

# => ["aaa", "bbb", "ccc", "ddd", "eee"]

example 3:

[1, "patayda", 2, "patayda", 3, "patayda", 4].map { |i| i.to_s }

# => ["1", "patayda", "2", "patayda", "3", "patayda", "4"]

example 4:

(1..4).map { "guau" }

# => ["guau", "guau", "guau", "guau"]

*note

#map works the same exact way that #collect works.

Also, if no block is provided when calling the method, an enumerator will be returned. So, for example, calling this:

[1, 2, 3, 4].map

will return this:

<Enumerator: [1, 2, 3, 4]:map>
